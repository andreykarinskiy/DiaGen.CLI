# 4. Стратегия решения

## 4.1. Ключевые принципы

### 4.1.1. Архитектурные стили
- CLI
- Microkernel (минимальное ядро, вся логика - в плагинах)
- Hexagonal Architecture

### 4.1.2. Принципы проектирования и дизайна
- SOLID
- BDD/TDD
- Dependency Injection

### 4.1.3. Важные паттерны проектирования
- Command - бизнес-логика плагинов
- Decorator - любые cross-cutting concerns, чтобы не засорять бизнес-логику
- Chain of Responsibility - для организации конвейнеров
- Composite - если некоторые команды составные

### 4.1.4. Технологический стек
- typer
- punq (DI)
- behave
- pytest

## 4.2. Архитектурные решения
- Утилита архитектурно состоит из ядра и плагинов:
  - ядро содержит логику загрузки и запуска плагинов, регистрации ошибок, а так же общие для плагинов контракты,
  - плагины содержат бизнес-логику.

- Плагины можно запускать или через DiaGen CLI, или напрямую, как отдельное консольное приложение.

- Ядро обнаруживает и загружает плагины динамически, во время исполнения.

- При ошибках используется стратегия FailFast.
