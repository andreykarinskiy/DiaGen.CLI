## 1. Введение и Цели

### 1.1. Концепция системы
Консольная утилита **DiaGen** (акроним от **Dia**gram **Gen**erator) предназначена для ускорения и упрощения (и другой автоматизации) задач подготовки технической документации, генерации и валидации моделей данных, контактов и других важных артефактов в IT-проектах.

Фактически, DiaGen - средство запуска различных Python-скриптов, выполняющих те или иные прикладные задачи. Эти скрипты можно запускать как по отдельности, так и вместе, выстраивая их в конвейеры, чтобы выполнять сложные комплексные задачи.
Новые скрипты добавляются в DiaGen с использованием pip.

### 1.2. Обзор требований

| №      | Требование                                                                                                                            | Мотивация и комментарии |
|--------|---------------------------------------------------------------------------------------------------------------------------------------|-------------------------|
| `FR01` | Выполнение скрипта отдельно по имени с входными параметрами                                                                           |                         |    
| `FR02` | Выполнение нескольких скриптов совместно (конвейер), где выходные параметры одних скриптов будут входными параметрами других скриптов |                         |    
| `FR03` | Подсказка (помощь) пользователю о DiaGen в целом (метаданные, версия) и обо всех доступных скриптах (их параметрах и версиях)         |                         |    

### 1.3. Цели качества

| №       | Цель                                                                      | Критерии | Мотивация и комментарии |
|---------|---------------------------------------------------------------------------|----------|-------------------------|
| `NFR01` | Детальная регистрация всех ошибок выполняющихся скриптов                  |          |                         |
| `NFR02` | Новые скрипты/функции создаются и добавляются максимально легко и просто  |          |                         |
| `NFR03` | Исходный код легко тестируется                                            |          |                         |
| `NFR04` | При любых сбоях утилита немедленно освобождает ресурсы и завершает работу |          |                         |
| `NFR05` | Простота использования в сценариях автоматизации (CI/CD, Docker)          |          |                         |
